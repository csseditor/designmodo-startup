Block for creating header. Codename “header-11”.

  1. Overview

    This block was developed as special header block. All blocks that marked as header should use once on one page. All header blocks have navigation bar. And any navigation bar has two states. Standard state –you can see now in the documentation. Second state – fixed-top state and you can see it for this block in the sample. Now the navigation bar is hidden, but you can activate it. For this you should remove class ".hidden" from this <header class="header-11 hidden">.

  2. Code

    Simple version:

    if you not already done all instructions that you had to read on the home page of this documentation please make it first For using this block you should only paste next rows of code into your html file: 

    include "../../../ui-kit-header/for-tpl/header-11.html"

    "LESS" version:

    if you not already done all instructions that you had to read on the home page of this documentation please make it first you should paste same code (as in simple version) and paste next code into your '.less' file after that rows that you already had to paste. 

    @import '../../ui-kit-header/less/header-11.less';

    All versions:

    So about modifications which you can do.

    This block, as many others header blocks in our framework, has special place for background image. The place for background image - <div class=”background”>...</div> For add your image as background you should add CSS rulers for this element. And you can manage opacity of this element. For example as this: 

    .background {
    background-image:url(‘your image url’);
    opacity: opacity value;
}

    In ‘Less’ version for opacity property you can use cross browser function: 

    .opacity(value from 0 to 100);

    This block has content which you can replace to yours. To make this you should replace content between tags <div class="span4">...</div>

    The block has special section for add video - <div class="player">...</div>.

    And yet this block has special functionality for sing-up, it realised as form <div class="signup-form"> <form>...</form></div> which data you can send to the server or do something else.
